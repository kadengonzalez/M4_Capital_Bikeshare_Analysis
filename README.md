Capital Bikeshare Ride Analysis
Overview

This project analyzes real-world bike-sharing data from Capital Bikeshare in Washington, D.C. using Python and Pandas. The goal is to identify patterns in ride behavior, including station usage, trip duration, and peak demand times.

The analysis explores how riders use the system across different days, hours, and user types (members vs casual riders). Insights from this data can help improve bike availability, optimize station placement, and better understand rider behavior.

Key Insights
Bike usage increases toward the end of the week, with Friday and Sunday being the busiest days
Peak ride times occur in the late afternoon (around 5 PM)
Members take significantly more rides than casual users
The median trip (10 min) is much lower than the mean (18.28 min), showing the data is skewed by long rides
